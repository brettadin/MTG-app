1. Overall Layout & Navigation

Global structure

Top bar

App name/logo.

Quick search (by card name).

Active data source indicator (e.g., “MTGJSON vX.X / Scryfall live”).

Icons: Settings, Theme toggle, Sync/Cache indicator.

Left sidebar navigation

Home / Search

Decks

Favorites / Collection

Analytics (optional / later)

Import / Export

About / Data sources

Main content area

Changes based on section.

Right-side context panel (card details) reused across sections.

High-level goals:

One search system reused everywhere.

One consistent card details panel (no matter if you’re searching, building a deck, or browsing favorites).

Deck building is always “one click away” from search results.

2. Key Screens
Screen 1: Card Search & Browser (Home)

Purpose: Find cards fast, filter them, view all printings, and add them to decks/favorites.

Layout

Top strip (under global header):

Big search bar

Placeholder: “Search by name, type, text, ‘flying’, ‘destroy target…’”

Advanced search toggle.

Filter row under search:

Format filter: [All | Standard | Pioneer | Modern | Legacy | Commander | Pauper]

Color identity filter: 5 color toggles (W, U, B, R, G) + colorless.

Mana value slider: min–max.

Card type chips: Creature, Instant, Sorcery, Artifact, Enchantment, Planeswalker, Land, Battle, etc.

Rarity checkboxes: C, U, R, M, S (special).

Set dropdown (with search).

Text contains box (for oracle text).

Price range slider (if you integrate prices).

Main content area:

Left: Results list/grid

Toggle: Grid view (images) / List view.

Each card tile shows:

Card art thumb.

Name + mana cost.

Type line (e.g., Legendary Creature — Goblin Artificer).

Set code + rarity.

Price (lowest market / average).

Small badge: “7 prints” for number of alternative printings/arts.

Quick actions:

[+] Add to deck (dropdown of decks).

[★] Favorite card.

Click tile → select card in details panel.

Right: Card Details panel

Large card image (current printing).

Tabs:

Info

Name, mana cost, type line, oracle text.

P/T or loyalty if relevant.

Legalities (pill tags: Standard, Modern, EDH, etc.).

Set / collector number, rarity.

Printings & Art

Gallery of all printings / arts as small thumbnails.

Click thumbnail → updates main image, shows that version’s price / set.

Favorite art button (for this printing only).

Prices

Table: vendor/source vs price.

Optional: price history sparkline.

Rulings

List of rulings (Plain text, scrollable).

Bottom of panel:

“Add to Deck” button (with deck picker).

“Add to Favorites” button.

“Pin to compare” (for multi-card compare later).

3. Component-Level Instructions (For Coding Agent)

You can hand this section almost verbatim to your coding assistant.

Core Components

CardSearchBar

Props: onSearch(queryObj), initialQuery.

Includes:

Text input (name or text search).

“Advanced” toggle to show filters.

Emits a structured query object:

{ name, text, colors, manaValueMin, manaValueMax, types[], rarities[], sets[], format, priceMin, priceMax }.

CardFilters

Reusable filter strip.

Props: value, onChange.

Used on Search screen and inside Deck Builder left column.

CardGrid / CardList

Displays an array of card summaries.

Each card summary object should contain:

id, name, manaCost, typeLine, setCode, rarity, imageUrl, price, printingCount.

Emits actions:

onSelect(cardId), onAddToDeck(cardId, deckId), onToggleFavorite(cardId).

CardDetailsPanel

Inputs: full card object + printing list + prices + rulings.

Tabs:

Info, Printings & Art, Prices, Rulings.

Actions:

onAddToDeck(cardId, deckId).

onFavoriteCard(cardId).

onFavoriteArt(printingId).

DeckListView

Inputs: deck object (cards with counts and zones: mainboard/sideboard).

Allows grouping by type / mana value / color.

Actions:

onChangeCount(cardId, newCount).

onRemoveCard(cardId).

onSelectCard(cardId).

DeckStatsPanel

Inputs: deck list with card details.

Renders stats:

Mana curve (counts by mana value).

Color distribution.

Types, land ratio.

Total price.

No complex AI needed initially; simple counts and charts.

FavoritesView

Uses CardGrid / CardList with source = favorite cards.

Additional filtering by tags and artist for Favorite Arts.

ImportExportView

Two subcomponents:

DeckImportForm: text area + parser results table.

DeckExportForm: deck selector + format selector + export actions.
